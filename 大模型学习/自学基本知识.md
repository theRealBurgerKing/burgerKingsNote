# Zero-shot 和 Few-shot

## Zero-shot（零样本）

**不给任何示例**，直接让模型完成任务，依赖模型从训练中获得的知识。

```
Prompt: 将下面的句子翻译成法语：
"Hello, how are you?"
```

模型直接回答，没有任何参考例子。

---

## Few-shot（少样本）

**提供少量示例**（通常 2~5 个），让模型学习输入输出的模式，再完成新任务。

```
Prompt:
英语 → 法语
"Good morning" → "Bonjour"
"Thank you" → "Merci"
"Hello, how are you?" → ?
```

模型通过示例"理解"你想要什么格式和风格。


# LLM 三个核心参数

## Temperature（温度）

控制输出的**随机性/创造性**。

- `0` → 确定性强，总选概率最高的词（适合代码、事实问答）
- `1` → 默认，较自然
- `>1` → 更随机、更有创意（适合创意写作，但可能乱）

---

## Top-p（核采样）

控制**候选词的范围**。

- 从概率累计达到 p 的词中采样
- `top_p=0.9` → 只从概率总和前 90% 的词中选
- 值越小 → 输出越保守集中
- 通常与 temperature **二选一调**，不建议同时大幅修改

---

## Max Tokens（最大长度）

控制**输出的最大 token 数**。

- 达到上限后强制截断
- 不影响内容质量，只限制长度
- 1 token ≈ 0.75 个英文单词 / 1~2 个汉字
## Stop Sequence（停止序列）

指定**让模型停止生成的特定字符串**。

- 遇到该字符串时立即停止输出
- 可设多个，如 `["\n", "END", "###"]`
- 适合结构化输出，防止模型"说多余的话"

```
# 例：只想要一行答案
stop: ["\n"]
```

---

## Frequency Penalty（频率惩罚）

根据**词已出现的次数**来惩罚重复。

- 出现越多次 → 下次被选中的概率越低
- 减少**重复用词**，让表达更丰富
- 范围通常 `0~2`，值越大惩罚越重

---

## Presence Penalty（存在惩罚）

只要**词出现过**就惩罚（不管出现几次）。

- 鼓励模型引入**新话题、新概念**
- 比 Frequency Penalty 更激进
- 范围通常 `0~2`


# CoT 和 ToT

## Chain of Thought（思维链）

让模型**一步步推理**，而不是直接给答案。

**做法**：在 prompt 加上 _"Let's think step by step"_ 或直接给出推理示例

```
Q: 小明有5个苹果，给了3个，又买了2个，还剩几个？

# 普通prompt → 直接答 "4个"
# CoT prompt → 
先算给出后：5-3=2
再算买入后：2+2=4
所以答案是4个
```

**优点**：显著提升复杂推理、数学、逻辑题的准确率 **缺点**：消耗更多 token

---

## Tree of Thought（思维树）

CoT 的升级版，模型**同时探索多条推理路径**，像树一样分支，最终选最优路径。

```
              问题
            /  |  \
          路径A 路径B 路径C
          ↓      ↓      ↓
         评估   评估   评估
                ↓
             最优解
```

**适合**：需要规划、搜索、多步决策的复杂任务（如谜题、策略问题）

**缺点**：计算成本高很多




## Reflection（反思）

让模型**评估自己的输出，发现错误，然后改进**。

```
第一次回答 → "答案是42"
      ↓
自我反思 → "等等，我的计算忽略了X条件"
      ↓
修正回答 → "答案应该是37"
```

**做法**：让模型输出后，再 prompt 一次：

> _"请检查你的答案，有没有错误或遗漏？"_

**适合**：代码生成、复杂分析、需要高质量输出的场景
